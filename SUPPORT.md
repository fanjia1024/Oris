# Support

## Where to ask for help

- Usage questions: open a GitHub issue using `Question / Support` template.
- Bug reports: open a GitHub issue using the bug template.
- Feature ideas: open a GitHub issue using the feature template.

## Before opening an issue

- Read [README.md](README.md) and examples under `crates/oris-runtime/examples/`.
- Check existing issues to avoid duplicates.
- Prepare a minimal reproducible example when possible.

## Security issues

Do not open public issues for vulnerabilities.

Follow [SECURITY.md](SECURITY.md) for private reporting.
